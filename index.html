poor module

A 15 line module system for node.js and the browser source on github

Poor-module lets you write the modules in node.js and the browser with minimal boilerplate code.

It's different from npm and other module systems
because it doesn't care about loading your files.

It assumes that your files are already loaded.
(Like you are used to in <script> tags)

Example use in node.js and the browser

Broswer node.js
Boilerplate
(index.html and app.js) 
// this file is index.html
<script src="poor_modules/poor_module.js" />
<script src="poor_modules/my_cool_module.js" />
<script src="main.js" />

// this file is app.js
require("poor_modules/poor_module.js")
require("poor_modules/my_cool_module.js")
require("main.js")
Your module code (say_hi.js) 
// this file is 'say_hi.js'
poorModule("say-hi", function () {
  return function () {
    console.log("hello world!")
  }  
})
The main file (main.js) 
// this file is 'main.js'
var sayHi = poorModule("say-hi")
sayHi();
run the file 
(visit http://yoursite.com/index.html in your browser)

node app.js
